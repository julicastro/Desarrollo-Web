Gulp

Permite tener multiples tareas y realizar diferentes acciones. Optimizamos imagenes, cambiar formatos, unir codigo de js, css. Necesitamos un workflow q automatice tareas.

Utilizamos npn i --save-dev gulp

Va a ser una dependencia de desarrollo al igual q la de sass

En la raiz del proyecto creamos el gulpfile.js q va a ser el contenedor de nuestras tareas

Se conecta package.json con gulpfile

Las tareas en gulpfile.js se llaman con npx gulp primerTarea

NPX se instala junto a node. Nos permite ejecutar paquetes sin necesidad de instalarlos globalmente

tambien podemos llamar a la funcion con npm. para eso debemos agregar lo siguiente al script del package.json:

"scripts": {"sass": "sass --watch src/scss:build/css", "tarea": "gulp tarea"},

Y luego puedo ejecutar con npm run tarea

Compilar SASS con Gulp

hago npm i --save-dev gulp-sass

Watch se ejecuta cuando hay cambios

Gulp Imagenes

Tranformar imagenes a webp q pesa menos

npm i --save-dev gulp-webp

Serie: todas las tareas se ejecutan de forma secuencial

Parallel: al mismo tiempo

Para reducir el tamaño de las imagenes instalamos lo siguiente: npm i --save-dev gulp-imagemin@7.1. esto me lleva a instalar otra dependencia para limpiar cache del procesamiento de imagenes. npm i --save-dev gulp-cache

Tendremos la misma imagen con diferentes formatos por si determinado navegador no lo soporta

Imagenes Avif

NUEVO FORMATO DE IMAGEN (2020)

npm i --save-dev gulp-avif

Hacemos lo mismo q con versionWebp:

const avif = require('gulp-avif')

function versionAvif(done) { const opciones = { quality: 50 // 0 a 100 }; src('src/img/**/*.{png, jpg}') .pipe(avif(opciones)) .pipe(dest('build/img')) done(); }

exports.dev = parallel(imagenes, versionWebp, dev);
